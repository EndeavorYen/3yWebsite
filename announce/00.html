參考手冊

一、基本配備﹕

1) 一台以 linux(v2.0.30 以上) 為作業系統的主機﹐CPU 486 即可﹐但記憶體最好32M以上﹐以釋放版本為例記憶體需求約為12M ﹐且 linux核心必須有 /proc 虛擬檔案的功能﹐但不需 root 的權限。

2) 最好具備網路功能。
3) 具有 tar(v1,12) 、 zip(v2.1) 、 gcc(v2.7.2.2) 等程式
4) 具有 crypt library（一般 linux 內附）

二、新手上路

我們假定你拿到的檔案名稱為 merc-fju.tar.gz ﹐你必須先在自己的目
錄下進行解壓縮

cd ~ --> 回到自身目錄
tar zxvf merc-fju.tar.gz > /dev/null --> 必須有 tar 和 zip

之後﹐就會產生一個目錄為 ~/mud﹐裡面的內容為﹕

angel/ 守護神資料目錄
command/ 所有指令的資料目錄
doc/ 之前 merc 的參考手冊
etc/ 雜項設定資料
include/ 參考資料（如職業、地形等等）目錄
player/ 玩家資料檔案目錄
src/ 原始程式碼
area/ 區域資料目錄
data/ 資料目錄
document/ 目前參考資料目錄
greeting/ 進站檔案目錄
joke/ 笑話集資料目錄
skill/ 技能資料檔案目錄
vote/ 投票資料檔案
board/ 版面設定以及資料目錄
debug/ 錯誤回報檔案
edit/ 之前 DOS 編輯程式目錄﹐但已不適用
help/ 線上求助檔案目錄
log/ 記錄檔案目錄
social/ 社交指令目錄
mail/ 信件目錄

如果配件不完全﹐請自行下載一份新的。

如果你的作業系統是 FreeBsd 則輸入
cp Makefile.bsd Makefile

之後進入 cd ~/mud/src 裡﹐編譯可執行檔﹐

make clean; make --> 必須有 gcc 和 make

以一台 Pentium 200 為例﹐編譯約需 5-7 分鐘﹐之後將會產生一個檔案
merc﹐如果沒有則代表編譯失敗﹐請至 140.136.196.3 bbs 問問看。因為釋放前﹐我們已經整理過了﹐所以一般狀況下﹐這裡不會出錯才對。

之後﹐必須修改 merc.ini 這個參考檔﹐一開始的重點為

1) NAME 自己遊戲名稱
2) MUD PORT 遊戲連接的埠
3) HOME DIRECTORY 遊戲的實際目錄

其餘的可以微調自己的系統。

整個都弄好之後﹐則可以到 cd ~/mud/src 裡執行

startup &

之後即可以開放連線﹐但必須注意的﹐因為裡面沒有任何玩家的資料﹐
所以第一個連線的玩家為超級玩家﹐也就是管理者﹐如果無法連線則可
以到 cd ~/mud/log 或是 cd ~/mud/debug 裡看看錯誤訊息﹐若有錯誤
也可以至 140.136.196.3 bbs 討論。

之後﹐遊戲就可以正式建立﹐但為了能讓遊戲能正常運作﹐所以我們釋
放了 6個區域目錄﹐我們希望能當正式運作後﹐除 limbo 以外﹐能夠
刪除﹐這一部份請參照版權宣告(COPYRIGHT)。

一開始﹐你必須先寫區域﹐你可以拿舊的區域檔案來做參考﹐這也是我們
為何要釋放一些區域的原因。每個區域都有幾個目錄﹕

index 區域的基本資料﹐必須存在
mineral/ 區域內礦物的設定
mob/ 虛擬怪物資料設定
obj/ 物品資料設定
res/ 重置設定
roo/ 區域房間設定
shp/ 區域內商店設定

之後就是﹐技能數值的設定﹐一般而言﹐武術以及閃躲技能都有數值﹐
分別為

1) Chance 出現機率
2) Value 攻傷數值

如果有開放的技能就必須從調整。
