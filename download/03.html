物品寫法介紹

一、檔案位置﹕

的物品就是由這些檔案所組成。

四、說明﹕

(1) Vnum (物品號碼)﹕
代表這個物品的虛擬號碼﹐這個選項不能被省略的﹐而且物品號碼
一定要是數字﹐且不可以小於 0﹐至於號碼的分配最好和區域裡的
房間號碼來對照﹐因為這樣比較方便清楚看出是哪一個區域的物品。

PS﹕物品與物品之間的號碼不可以相同﹐與區域或是怪物所用的號
碼則可以重複。

(2) Name (物品英文名稱)﹕
代表物品的英文名字﹐他是代表物品的最短名字﹐所有的動作都和
這個名字有關﹐請以字串結束符號( ~ ) 結束﹐這個選項不可以被
忽略﹐這一個字串最好是英文﹐因為若不是英文會照成玩家的不便。

(3) ShortDesc (物品中文描述)﹕
這個字就是當你觀看自己或是別人身上攜帶的物品時所顯示的簡短
敘述﹐他也在程式裡 act 函數和其他函數當作辨識物品所用到的
字串﹐請以字串結束符號( ~ ) 結束。這個選項是不可以被忽略﹐
這一個字串最好可以包含 Name 字串裡面的英文名字﹐這樣對於辨
識上比較方便。

(5) Description (物品描述)﹕
若是某一個人物進入一個房間的時候﹐而且這個人可以看到這樣物
品的話﹐那麼就會顯示出這一段訊息﹐因為考慮到對齊﹐所以是從
這個關鍵字下一行開始撰寫﹐上一行所有的東西都將被忽略﹐請以
字串結束符號( ~ ) 結束﹐這個選項不可以被忽略﹐這一個字串最
好可以包含 Name 字串裡面的英文名字﹐這樣對於辨識上比較方便。

(6) Wearmsg (裝備敘述)﹕
這個字串是當你裝備物品的時候整個房間可以看的到的敘述﹐敘述
都是在關鍵字後一行才開始算﹐避免讀檔時空白被吃掉﹐請在第一
個字元寫上 '.' 這個符號﹐才不會被吃掉﹐不過這個符號是不會
顯示出來的。

(7) Remmsg (移除裝備敘述)﹕
這個字串是當你移除身上裝備物品的時候整個房間可以看的到的敘
述﹐敘述都是在關鍵字後一行才開始算﹐避免讀檔時空白被吃掉﹐
請在第一個字元寫上 '.' 這個符號﹐才不會被吃掉﹐不過這個符
號是不會顯示出來的。

(8) Unit (物品單位)﹕
這個字串是代表這個物品的單位﹐例如說一個饅頭﹐一把長劍等等
﹐如果你沒有寫的話﹐預設是以個為基本單位。

個別說明如下﹕

□閃閃發光(GLOW)﹕
若物品有這個旗標的話﹐會在物品前面附上 (閃閃發亮) 的符
號。

□嗡嗡作響(HUM)﹕
若物品有附上這個旗標﹐會在物品前面加上 (嗡嗡作響) 的符
號﹐這個要小心使用﹐不然到時候會出現嗡嗡作響的饅頭。

□黑暗(DARK)﹕
□保留(LOCK)﹕
這兩個旗標目前保留﹐不使用。

□邪惡(EVIL)﹕
這個旗標讓這個物品是邪惡的﹐並且用偵測邪惡的法術可以看
出。

□隱形(INVIS)﹕
這個旗標使得這個物品隱形﹐平常是看不到的﹐也不能裝備脫
掉等等﹐要使用偵測隱形的法術才可以看到。

□魔法(MAGIC)﹕
表示這個東西是魔法物品﹐即附有魔法的物品。

□不可丟棄(NODROP)﹕
設定這個物品不可以丟棄﹐當然連一些拍賣﹐或是賣掉都不行。

□祝福(BLESS)﹕
表示這個物品有受過祝福。

□抗善良陣營(AntiGood)﹕
設定這個物品善良陣營的人不能裝備﹐所謂善良陣營就是陣營
在 +300 以上的人。

□抗邪惡陣營(AntiEvil)﹕
設定這個裝備邪惡陣營的人無法裝備﹐邪惡陣營就是陣營在
-300 以下的人。

□抗中立陣營(AntiNeutral)﹕
這個旗標設定這個裝備讓中立陣營的人無法裝備﹐中立陣營是
陣營值在 +300 到 -300 之間的人。

□不可解除裝備(NOREMOVE)﹕
設定這個裝備如果裝備上去的話沒有辦法停止裝備。

□死後消失(INVENTORY)﹕
如果物品有設定這個旗標的話﹐當主人死掉這一項物品便會消
失﹐一般來說﹐商店的物品都已經自動設定這一個旗標了。

□可以封印魔石(CANCACHET)﹕
設定這一個裝備可以封印住魔石﹐不過如果這物品不是可以裝
備的話也是不能封印的。

□自設數值(USERSET)﹕
這個旗標目前不使用﹐如果想要給系統設定攻擊傷害或是防禦
力的話﹐請將該項數值設為小於零即可。

□不儲存(NOSAVE)﹕
如果設定這個旗標的話﹐這個物品不會被儲存在檔案裡﹐像是
一些鑰匙或是覺得不要馬上開放的東西﹐便可以設定這個旗標
﹐等待已經可以開放的時候﹐在將這個旗標拔除即可。

□違禁品(CONTRABAND)﹕
這個旗標設定這個物品是武鬥大會的違禁品﹐即在武鬥大會之
前會沒收的物品﹐如果你覺得這個物品會影響到武鬥大會的公
平性﹐你便可以加入這個旗標﹐但是請注意﹐不要亂加﹐還有
藥水或藥丸類一律加入。

□不可偷竊(Steal)﹕
辛辛苦苦解迷出來的東西被隨隨便便就偷走了，是不是很不合
理呢? 可以設定這個旗標使這樣物品無法偷竊。

□不可拍賣(Auction)﹕
這是設定成無法拍賣的物品，如解迷物品、或是神族的裝備。

□繫緊(Hook)﹕
這是設定成繫緊的狀態。

(10) Weight (物品重量)﹕

有很大的關係﹐你可以省略這個選項﹐系統的內定值是這樣物品的
等級乘以十。

(12) Level (物品等級)﹕
代表這個物品的等級﹐這是一個數字﹐可以被省略不寫﹐內定值是
1。

此參數設定這個物品在一些丟棄的情況下是否會消失﹐妳可以用字
串常數來代替數字﹐當你設定物品會消失( YES ) 的時候﹐當這個
物品被丟棄( drop ) 到地上的時候便會消失掉。

(15) Armor (物品強度)﹕
這數值設定這項裝備的強度﹐可以抵抗多少的攻擊﹐當然如果不是
裝備的話這一項數值是沒有用的﹐如果妳沒有給定的話系統會有預
設﹐如果妳要自行設定的話﹐你可以設定為等級的一百倍即可。

(16) Exp (物品升級經驗)﹕
這一項數值設定物品是否可以升級與升級的經驗值﹐以一萬為準﹐
只有武器或是防具可以升級﹐要升級的話必須設定此項數值﹐請勿
設定太高﹐以三百為限﹐即物品的升級經驗值以萬分之三百為限制
﹐若是超過會變的非常容易升級。

(4) 物品威力與防禦程度設定:

☆ 重要!! 請依照本設定來設定數值!! ☆

物品中有需要調整的地方有 重量, 等級, 價格, 損壞, 跟各項數值.

1. 重量 (weight)﹕
基本上依照自己的想像力去設定就好﹐不過大致上來說﹐武器類
不要設定的太輕﹐看看材質來設定﹐如木頭做的就比較輕一點﹐
鐵做的就重一點﹐而防具部份則大概依照大小來設定就可以了﹐
平均值為等級 / 5 取整數即可﹐以這個數字來浮動。

2. 等級 (level)﹕
這個數值基本上就非常依照妳自己的條件來設定了﹐不過太普通
的東西一樣不要設定太高的等級﹐相同的﹐很好的東西請不要設
定很低的等級﹗


3. 價格 (cost)﹕
一樣物品的價格基本上程式是設定為 等級的乘以十 去浮動﹐
妳如果有需要更動的話﹐也請不要超過這一項數值太多或是太少
﹐如果超過系統會記錄並要求更改。

4. 損壞 (armor)﹕
這一項就是裝備的損壞率﹐以等級乘以一百為準。

5. 各項數值 (value)﹕
有需要動到的只有武器與防具兩種。
☆傷害力的設定﹐依照武器的種類要有不同的設定﹐就算是在強
的武器﹐也不能將傷害力設定得太誇張﹐武器的強度設定如下﹕

斧 > 槌 = 棍 > 刀 > 劍 > 拳 = 槍 > 鞭 > 杖 = 弓 > 筆 = 匕首

傷害力基本算法一一百起跳﹐最大傷害再加上等級的一半﹐最
後一武器的不同加上額外傷害力( 斧加七 )﹐所以是不會有兩
百以上傷害的武器出現﹐也請大家不要寫出傷害兩百以上的武
器﹗
☆防具防護的等級﹐如果設定 -1 則由系統自動給予﹐標準是每
二十級加一個防護等級﹐所以如果你自己設定防護等級的話﹐
請勿超過這個範圍太多。

6. 附加數值(affect)﹕
每一個物品所附加的數值其中以加屬性為最需要討論﹐基本上如
果要加強各項屬性的話﹐也請不要只有加而沒有減﹐這樣基本上
玩家會太強﹐並且加強的話單項以加三為準﹐請勿超過﹐總和不
能超過二﹐如果等級減半﹐則屬性也要更低﹗至於其他如命中率
等等﹐則以一百為限。

☆ 此處設定隨實有可能變更, 請多注意版上文章.
